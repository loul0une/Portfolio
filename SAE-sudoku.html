La demande

Il s'agissait de réaliser un jeu de sudoku fonctionnel de taille variable. Les grilles étaient chargées à partir de fichiers fournis.

J'ai réalisé ce projet seul en langage C. J'ai utilisé un fichier de grilles de sudoku pour les charger dans le programme. J'ai ensuite créé une fonction pour vérifier si la grille est valide. J'ai également créé une fonction pour vérifier si la grille était valide.

Étapes de travail

1 : Phase de maquettage : Présenter notre maquette du projet à notre client et exposer à quoi allait ressembler notre sudoku et comment il allait fonctionner.

2 : Phase de conception algorithmique : Concevoir l'algorithme qui va permettre le bon fonctionnement du jeu de sudoku. Puis réaliser le cahier de test concernant les résultats potentiels du programme.

3 : Phase de développement : Coder l'algorithme et le tester puis entrer les résultats dans le cahier de test. Mais également une documentation détaillée des fonctions et du programme à l'aide de Doxygen.

4 : Phase de modification : Modifier le programme en moins d'une heure, à la demande du client.
